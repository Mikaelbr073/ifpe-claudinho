# SisVagas UBS

A web-based appointment scheduling system for public health clinics (UBS — Unidade Básica de Saúde), built as a demo project for a live talk at **IFPE (Instituto Federal de Pernambuco)** on AI-assisted software development.

---

## About the Project

**SisVagas UBS** centralizes and optimizes appointment slot management in public health clinics — reducing physical queues, no-shows, and idle time in medical schedules.

The system is split into three modules:

- **Citizen** — online booking and cancellation of appointments
- **Reception / Health Professionals** — check-in panel, daily schedule, and time-slot blocking
- **Management / Regulation** — slot grids, smart waitlist, and occupancy reports

For full requirements, personas, and KPIs, see [PRD.md](./PRD.md).

## IFPE Talk — Claude Code in Practice

This repository was used live during a talk at IFPE to demonstrate how **Claude Code** — Anthropic's AI CLI — integrates into a real product development workflow.

What was shown in practice:

- AI-assisted requirement writing and PRD refinement
- Next.js project scaffolding with architecture rules defined in `CLAUDE.md`
- Code style, security, and workflow rules read automatically by the agent
- Feature development with code review via a `code-reviewer` subagent
- Conventional Commits messages generated by the `/commit` skill

> The goal was to show that AI doesn't replace the developer — it acts as a collaborator that knows the project, follows the team's rules, and speeds up repetitive tasks.

## Running Locally

> **This project runs fully mocked — no environment variables or external services are required.**
> Just install dependencies and start the dev server. Database, auth, and external integrations are all stubbed out for local development.
>
> Environment variables and real service connections are only needed if you want to deploy the app or evolve it beyond the demo.

### 1. Clone and install

```bash
git clone https://github.com/Mikaelbr073/ifpe-claudinho.git
cd ifpe-claudinho
npm install
```

### 2. Start the dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

---

## Going Beyond the Demo (optional)

If you want to connect real services and deploy the app, create a `.env.local` file based on `.env.example` and fill in your credentials:

```env
DATABASE_URL=postgresql://user:password@localhost:5432/sisvagas
NEXTAUTH_SECRET=your-secret-key
NEXTAUTH_URL=http://localhost:3000
GOOGLE_CLIENT_ID=your-client-id
GOOGLE_CLIENT_SECRET=your-client-secret
```

Then run the database migrations:

```bash
npx prisma migrate dev
```
